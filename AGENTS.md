# AGENTS.md

## Product UX Review

When the user asks for a product review, UX review, regional UX review, target-audience review, or best-practice review, use the instructions in `product_ux_review.md`.

Before changing code, first produce a structured product UX review.

For user-facing changes, evaluate:

- target audience fit;
- regional and cultural fit;
- accessibility;
- copy clarity;
- onboarding friction;
- empty/error/loading states;
- mobile behavior;
- analytics needed to validate the change.

If the requested task is a PR/code review, use these product UX review instructions in addition to normal code review.
